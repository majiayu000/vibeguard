# Security Disclosure Policy

Security vulnerabilities must not be handled in public issue or pull request
threads unless a maintainer has explicitly decided disclosure is safe.

## Public Workflow Behavior

- Label public reports as `security_private` when a report may disclose a
  vulnerability.
- Ask the reporter to use the repository's private security channel.
- Do not ask agents to reproduce secrets, exploit steps, or private data in
  public comments.

## Agent Boundary

Agents may identify possible security sensitivity and recommend escalation.
Agents must not make the final disclosure decision.
