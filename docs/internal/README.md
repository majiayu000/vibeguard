# Internal Notes

This directory contains VibeGuard material that helps maintainers understand past decisions but is not part of the public runtime or install contract.

## Layout

| Path | Role |
|------|------|
| `research/` | Knowledge discovery notes, external article mirrors, and follow-up research. |
| `history/` | Historical design snapshots that no longer define current implementation behavior. |
| `benchmarks/` | Benchmark design notes and evaluation planning. |
| `plans/` | Cleanup and architecture plans that should not be confused with active `plan/` workflow output. |
| `followups.md` | Cross-session follow-up index for maintainer work. |

For public repository layout guidance, see `docs/directory-map.md`.
