## 知识摘要

### 1. Claude Code System Prompt 组装架构（19 分）
- **来源**: [How Claude Code Builds a System Prompt](https://www.dbreunig.com/2026/04/04/how-claude-code-builds-a-system-prompt.html)
- **核心**: Claude Code 通过 30+ 条件组件动态组装 system prompt，而非静态字符串
- **关键发现**:
  - 三层压缩策略：函数结果清理（保留最近 5 个）→ 摘要要求（agent 必须主动记录关键信息）→ Microcompact 自动清除过期输出
  - `__SYSTEM_PROMPT_DYNAMIC_BOUNDARY__` 缓存边界标记，分离全局可缓存内容与每轮会话数据，避免静态指令重复计算
  - 渐进式工具暴露（graduated tool disclosure）：~50 工具按条件可见，非全量暴露
  - 风险分级确认：本地可逆操作自由执行，不可逆/共享操作需用户确认
- **VG 关联**: 验证 VibeGuard L1-L7 分层设计的合理性；三层压缩与我们的 Compaction 保留清单互补

### 2. CodeScene: Code Health MCP 服务 + Agent 质量门禁（18 分）
- **来源**: [Agentic AI Coding: Best Practice Patterns](https://codescene.com/blog/agentic-ai-coding-best-practice-patterns-for-speed-with-quality)
- **核心**: AI agent 操作代码前先测量 Code Health，低于 9.5/10 不允许 agent 修改
- **关键发现**:
  - 三层自动检查点：生成时连续 review → 预提交验证 → PR 预检
  - AGENTS.md 行为合约：无文档化原则时 agent "invoke tools opportunistically"（机会主义调用），导致防线弱化
  - 覆盖率作为行为回归信号（非虚荣指标）：防止 agent 删除测试作为"捷径"
  - 遗留代码函数太大时 agent 错误率飙升 → 重构优先于 agent 部署
- **VG 关联**: 强化 U-22（测试覆盖率）和 W-12（测试完整性保护）；AGENTS.md 概念与我们的 CLAUDE.md 规则体系一致

### 3. 三层 Agentic Guardrails 实现（16 分）
- **来源**: [FareedKhan-dev/agentic-guardrails](https://github.com/FareedKhan-dev/agentic-guardrails)
- **核心**: Input（异步并行验证）→ Action Plan（执行前拦截）→ Output（幻觉检测 + 引用验证）
- **关键发现**:
  - Layer 1 用 asyncio 并行运行 3 个检查（主题/PII/威胁），任一失败立即拒绝
  - Layer 2 提取 agent 的"行动计划"独立验证，可疑操作升级为人工审批
  - Layer 3 幻觉检测：snippet matching + context window + source attribution 三重验证
  - 模型分层策略：Gemma-2-2B（快速高吞吐）/ Llama-Guard-3-8B（安全分类）/ Llama-3.3-70B（复杂推理）
  - 先展示无防护 agent 的灾难性失败，再逐层添加保护
- **VG 关联**: 与 VibeGuard L1-L7 架构高度对齐；"action plan guardrails" 映射到 preflight 概念

### 4. Agentic Engineering PEV Loop（15 分）
- **来源**: [NxCode: Agentic Engineering Complete Guide](https://www.nxcode.io/resources/news/agentic-engineering-complete-guide-vibe-coding-ai-agents-2026)
- **核心**: Plan → Execute → Verify 循环替代 vibe coding 的 "prompt and hope"
- **关键发现**:
  - "计划阶段是工程专业知识最重要的环节。模糊计划产出模糊代码。"
  - 多 agent 角色分工：Feature Author / Test Generator / Code Reviewer / Architecture Guardian / Security Scanner
  - AI Slop 定义：表面合理但缺少错误处理、引入安全漏洞、破坏现有功能的代码
  - 安全扫描嵌入 PEV 循环内部，而非事后 bolt-on："1000 PRs/week × 1% 漏洞率 = 每周 10 个新漏洞"
- **VG 关联**: PEV 映射到 VibeGuard 的 preflight → check → review 流程，验证设计方向

### 5. Claude Mythos Preview + Project Glasswing（15 分）
- **来源**: [Anthropic Red Team: Mythos Preview](https://red.anthropic.com/2026/mythos-preview/)
- **核心**: SWE-bench 93.9%，自主发现数千个 0-day 漏洞，因安全风险不公开发布
- **关键发现**:
  - Firefox 181 个 exploit（vs Opus 4.6 的 2 个），包括 27 年历史的 OpenBSD TCP SACK 漏洞
  - 能力涌现：非专门训练安全能力，而是代码/推理/自主性通用提升的下游结果
  - "基于摩擦而非硬性屏障的安全缓解措施在模型辅助攻击者面前将显著弱化"
  - Project Glasswing：12 家科技巨头联合防御性使用，SHA-3 加密承诺公开
- **VG 关联**: 双用途加速警告 — 同样的改进既增强修补能力也增强利用能力；强化 SEC-11 的 AI 代码安全审查必要性

### 6. MCP v2.1 Server Cards + 发现机制（14 分）
