## 高价值发现

### 1. Martin Fowler — Context Anchoring
**Insight Card**: `[旧共识: AI 对话状态靠上下文窗口维持] vs [新发现: 应外化到 feature-level 文档，使决策"survive session boundary"]`

- 核心机制：把"为什么选 X（以及拒绝了 Y）"写入随 session 启动加载的持久文档
- 里程碑式 litmus test：**"你能否无焦虑地关闭 session？"** — 如果不能，决策困在错误媒介
- 对比："Lost in the Middle" 研究表明长 context 中间段性能显著下降，锚定文档避免这个退化
- **与 VibeGuard 的关系**：与 MEMORY.md / exec-plan.md 完全同构；litmus test 可以纳入 exec-plan 验收标准
- 来源：https://martinfowler.com/articles/reduce-friction-ai/context-anchoring.html

### 2. Anthropic — Code Execution with MCP
**Insight Card**: `[旧共识: MCP 把工具 schema 灌进 context] vs [新发现: 让 agent 写代码、按需读 tool 定义，可 98.7% 降 token]`

- 具体数字：150K → 2K token（一个连接数千工具的场景）
- 中间结果问题：2h 会议转录从 Google Drive 到 Salesforce，走 model 两次 → 多 50K token
- **PII Tokenization**：MCP client 拦截数据，替换为 `[EMAIL_1]`，仅在目标工具被 untokenize；真实 PII **不经过模型**
- 架构启示：下一代 agent 更像"写代码调编排"，少像"直接 API client"
- 来源：https://www.anthropic.com/engineering/code-execution-with-mcp

### 3. arXiv 2506.01333 — MCP Tool Squatting / Rug Pull (ETDI)
**Insight Card**: `[旧共识: MCP 工具安装后可信] vs [新发现: 工具描述可静默改写 → rug pull，需加密签名 + 不可变版本 + policy engine]`

- 验证了 SEC-12 已记录的 4 类攻击（Tool Poisoning / Rug Pull / Tool Shadowing / Direct Injection）
- **新补充**: ETDI 方案给出三层防御：**加密身份验证 + 不可变注册表 + OAuth 2.0 + 策略引擎**
- 静态 OAuth scope 不够：需 runtime 上下文感知策略
- 来源：https://arxiv.org/abs/2506.01333

### R2.1 Aider — Architect/Editor 模型分离
**Insight Card**: `[旧: 单模型包打天下] vs [新: 推理模型（o1-preview）写解决方案 + 编辑模型（Sonnet/DeepSeek）翻成 diff → 85% SOTA, 14× 降成本]`

- 来源：https://aider.chat/2024/09/26/architect.html

### R2.2 Addy — The 80% Problem / Comprehension Debt
**Insight Card**: `[旧: AI 代码跑通就合入] vs [新: 你不理解的合入 = 复利负债，inverted effort 70% 问题定义+验证 / 30% 执行]`

- **Comprehension debt（Jeremy Twei 命名）**: agent 不累你会累→ 评审压力下推未真正理解的代码上线 → 未来无法维护
- 最后 20% 的内容：假设校验 / 概念 pushback / 生产加固 / 架构一致性
- 5 条战术：
  1. **反转 70/30**: 70% 问题定义+验证策略，30% 执行
  2. **Fresh-context self-review**: 让模型用干净 context 批判自己的输出
  3. **Tests first** 作为客观成功标准
  4. **保留手写练习** 防问题解决肌肉退化
  5. **承认心理冲击**: orchestrator 角色感觉像管理
- 数据：团队 PR +98% 但 review +91% → 认知负荷只是从写转移到审
- 来源：https://addyo.substack.com/p/the-80-problem-in-agentic-coding

### R2.3 Simon Willison — MCP Prompt Injection（通俗版）
**Insight Card**: `[旧: MCP 独有安全问题，可通过技术方案解] vs [新: 这是 2.5 年未解的通用 prompt injection，Willison 承认"不知道怎么解"，只能靠 UI + human-in-loop]`

- 三类攻击：
  1. **Tool Poisoning**: 工具描述埋指令读 `~/.cursor/mcp.json` 外泄
  2. **Rug Pulls**: 安装后静默改描述重定向 API key
  3. **WhatsApp exploit**: 假"日常小知识"工具用空白符混淆转发消息历史
- 客户端防御：首次展示工具描述 / 描述变更告警 / 不用隐藏滚动条
- 服务端防御：sanitize 外部输入 / 不 shell 调 unsanitized / UI 清晰
- **关键分歧**: Willison 拒绝给技术解决方案，"I don't know what to suggest"——与 ETDI 学术解不同
- 来源：https://simonwillison.net/2025/Apr/9/mcp-prompt-injection/

### R2.4 addyosmani/agent-skills（GitHub 17K⭐）
